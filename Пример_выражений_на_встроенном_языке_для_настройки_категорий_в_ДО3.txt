По автоматическому присвоению категории «ВИП»/VIP для документов по контрагенту с установленным признаком «ВИП»/VIP (через группу доступа)

Запрос = Новый Запрос;
Запрос.Текст = 
"ВЫБРАТЬ
|	ДокументыПредприятия.Контрагент КАК Контрагент,
|	ДокументыПредприятия.Ссылка КАК Ссылка
|ИЗ
|	Справочник.ДокументыПредприятия КАК ДокументыПредприятия
|ГДЕ
|	ДокументыПредприятия.Контрагент.ГруппаДоступа = &ГруппаДоступа
|     И ДокументыПредприятия.Контрагент В (&Контрагент)
|";
Запрос.УстановитьПараметр("ГруппаДоступа", Справочники.ГруппыДоступаКонтрагентов.НайтиПоКоду("00-000001")); 
Запрос.УстановитьПараметр("Контрагент", Источник.Контрагент);
Выборка = Запрос.Выполнить().Выгрузить();
Если Выборка.Количество()>0 Тогда
Результат = Истина;
КонецЕсли;  


По автоматическому присвоению категории «ВИП»/VIP для документов по контрагенту с установленным признаком «ВИП»/VIP (через доп. реквизит) (скрипт из Интернета)

- доп. реквизит необходимо добавить для всех видов контрагентов
- при создании необходимо выбрать реквизит Из другого набора, т.е. использовать ранее созданный
- не забывайте, что обращение к реквизиту происходит через его "Имя" (наименование для разработчика)
- можно доп. учесть ситуацию, когда в документе разные контрагенты с разными статусами 

Статус = ПланыВидовХарактеристик.ДополнительныеРеквизитыИСведения.НайтиПоРеквизиту("Имя", "НаименованиеДляРазработчика");

Если Не ТекСвойство = Неопределено Тогда Результат = ТекСвойство.Значение; Иначе Результат = Ложь;
